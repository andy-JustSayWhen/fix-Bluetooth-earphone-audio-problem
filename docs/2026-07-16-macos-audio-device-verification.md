## 核对目的

## 实时系统查询

使用 macOS 自带的 `system_profiler SPAudioDataType -json` 查询当前音频设备状态。

| 路由 | 当前设备 | 连接类型 | 采样率 | 查询依据 |
| --- | --- | --- | --- | --- |
| 普通音频输出 | Redmi 电脑音箱 | USB | 48 kHz | `coreaudio_default_audio_output_device: spaudio_yes` |
| 音频输入 | Bose QC Headphones | 蓝牙 | 16 kHz | `coreaudio_default_audio_input_device: spaudio_yes` |
| 系统提示音输出 | Bose QC Headphones | 蓝牙 | 44.1 kHz | `coreaudio_default_audio_system_device: spaudio_yes` |

这台电脑当前不是“所有声音都走同一个设备”：普通播放声音走 Redmi 电脑音箱，麦克风输入走 Bose QC Headphones，系统提示音走 Bose QC Headphones。

## 一致性结论

| 对比项 | 实时系统状态 | 界面显示 | 是否一致 |
| --- | --- | --- | --- |
| 普通播放输出 | Redmi 电脑音箱 | 系统设置“输出”高亮 Redmi；顶部菜单栏输出高亮 Redmi | 是 |
| 音频输入 | Bose QC Headphones | 顶部菜单栏输入高亮 Bose | 是 |
| 系统提示音输出 | Bose QC Headphones | 系统设置顶部“播放声音效果的设备”显示 Bose | 是 |
| 系统设置中的输入高亮项 | Bose QC Headphones | 提供的截图没有展示“输入”标签页内容 | 无法从截图直接核对 |

因此，本次没有发现“系统实际设备”和两个前端界面显示不一致的问题。看起来不一致的原因是系统同时存在三个独立路由：普通播放输出、音频输入、系统提示音输出。

## 对蓝牙耳机音质异常排查的意义

Bose QC Headphones 当前承担音频输入，而且输入采样率是 16 kHz；它同时承担系统提示音输出，但普通播放输出实际上是 USB 的 Redmi 电脑音箱。若某个应用在播放音频时又启用了 Bose 的麦克风，应用可能会改变自己的输出路由或触发蓝牙通话模式。此记录只能证明当前设备路由和界面显示是一致的，不能单凭它证明某个应用一定已经进入通话模式。

后续遇到“声音空洞、失真、码率很低、传输不稳定或没有声音”时，应同时记录：

1. 系统普通播放输出是否仍为目标蓝牙耳机。
2. 系统输入是否启用了目标蓝牙耳机麦克风。
3. 出问题的应用自己的输入、输出选择。
4. 出问题瞬间目标蓝牙设备的采样率和连接状态。
