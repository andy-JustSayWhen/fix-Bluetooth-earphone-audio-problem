# 耳机处于 A2DP、音频可播放但耳机端没有声音的解决办法

## 结论

当系统已确认耳机处于 A2DP、播放应用已启动输出、Mac 也在持续发送蓝牙音频，但耳机端没有声音时，可以尝试断开并重新连接该蓝牙耳机。

这是本项目已实测有效的方法之一。它通过重建 Mac 与耳机之间的蓝牙音频连接恢复播放，属于临时恢复方法，不等于已确认根因。

## 适用条件

在执行重连前，至少应确认：

- 目标耳机为 A2DP 等高音质播放模式，不是 HFP 等低音质语音模式。
- 目标耳机的输出端点具有高于 `16 kHz` 的实际采样率和不少于两个输出声道，或最新声音链路为 `tacl`。
- 播放应用确实已向该耳机启动输出，不是仅界面显示正在播放。
- Mac 端的该输出端点没有静音，且音量不为零。
- 音频在其他内置或有线输出设备上可正常播放。

如果目标耳机实际处于 HFP，应按 [HFP 模式原因与恢复路由](蓝牙音频设备进入HFP模式的原因.md) 处理，不应把本方法冒充为 HFP 一键修复。

## 操作方法

1. 保留当前播放现场，记录目标耳机、播放应用、模式、链路、采样率、声道、音量和静音状态。
2. 只断开目标蓝牙耳机，不删除配对记录，也不断开其他蓝牙设备。
3. 等待该耳机在 macOS 中确认断开后，重新连接它。
4. 确认耳机重新出现，并复核实际模式、声音链路、采样率和声道。
5. 让原播放应用继续或重新向该耳机输出，最终以用户确认耳机实际恢复发声为验收条件。

## 2026-07-21 K03S 实测

- 宿主机：`andy的macbook air`；MacBookAir10,1；Apple M1；macOS 14.6.1 / 23G93。
- 目标设备：XIBERIA K03S，蓝牙地址 `50:C0:F0:F3:6A:66`。
- 播放应用：QQ音乐 9.2.5。
- 故障现象：QQ音乐显示正在播放，切到其他有线扬声器时有声，切到 K03S 时耳机无声。
- 模式证据：K03S 为 `A2DP / tacl / 44.1 kHz / 2 声道`，不是 HFP；输出音量为 `81.25%`，未静音。
- 播放证据：系统明确记录 QQ音乐在 K03S 的 `tacl` 输出上启动播放，耳机回复接受 A2DP 播放流，Mac 持续发送每秒约 `256 千比特` 的音频数据。
- 无效方法：重启 `audioaccessoryd`（macOS 负责蓝牙音频路由决策的系统进程）后，耳机仍然无声。
- 有效方法：05:25 只断开并重新连接 K03S；重连后系统再次确认 `A2DP / tacl / 44.1 kHz / 2 声道`，QQ音乐重新申请 K03S 输出并启动 A2DP 音频流，用户确认耳机恢复声音。

## 证据边界

- 本次可以确认“重连 K03S 之后恢复发声”，不能确认是另一台已连接主机抢占了播放权、耳机内部播放状态卡死，还是其他耳机端原因。
- macOS 已经建立 A2DP 播放流并持续发送数据，因此故障位置已经缩小到 Mac 输出链路之后；但 macOS 日志无法直接观察耳机内部如何选择音源和驱动扬声单元。
- 本次是同一台 Mac、同一台 K03S 和同一个播放应用的一次实测，尚不能推广为所有蓝牙耳机都能通过重连恢复。
- 重连会中断目标耳机的现有音频流，播放应用需要自动重申请或由用户重新选择该输出。
