# 传统蓝牙麦克风导致播放质量下降

## 结论

对传统蓝牙音频而言，耳机只做播放时通常使用 A2DP（蓝牙立体声播放模式）；当应用打开同一耳机的麦克风时，系统需要双向传输，传统链路会使用 HFP（蓝牙通话模式）。HFP 同时提供麦克风输入和单声道播放，常见语音采样率为 8 kHz 或 16 kHz，因此音乐可能变成空洞、失真、低码率，也可能在模式切换或应用重新建立音频流期间短暂无声。

这不是“耳机损坏”的直接证据，也不能仅凭听感判断具体编码器。可靠判断应同时记录：当前输入和输出端点、采样率、声道数、系统日志中的传输配置，以及应用音频流是否正在重建。

## 直接证据

### Apple 对 macOS 现象的描述

Apple 明确写明：播放音乐或其他音频时，如果打开使用蓝牙耳机内置麦克风的应用，音质和音量会降低，也可能出现静电声或爆音。Apple 将其解释为蓝牙存在两种模式：一种用于高质量聆听，另一种同时用于麦克风说话和聆听；切到第二种模式后，音质会降低，直到麦克风不再使用。

来源：[Apple 原文](../raw/apple/apple-macos-bluetooth-headphones-sound-quality.md)。

### Microsoft 对传统蓝牙协议的定义

Microsoft 明确区分：

- A2DP 支持高质量立体声播放，只负责主机到耳机的输出，不提供耳机麦克风采集。
- HFP 支持麦克风采集和单声道播放，分为窄带 8 kHz 和宽带 16 kHz 两种常见模式。
- Windows 10 中，应用打开免提麦克风后，设备进入 HFP，送往立体声端点的音频会被丢弃。
- Windows 11 中，输入端点被打开或通信类播放流建立时，系统选择 HFP；其他情况下使用 A2DP。系统会把其他应用的音频重采样到当前 HFP 端点的格式。

来源：[Microsoft 传统蓝牙音频原文](../raw/microsoft/bluetooth-classic-audio.md)。

### Microsoft 对通信场景的补充

Microsoft 进一步说明，蓝牙设备在通信场景下可能改变采样率、声道数等格式；传统蓝牙使用 HFP 时，输入和输出端点都为单声道，因此传统蓝牙设备在麦克风使用期间不能保持立体声播放。Windows 11 支持 CVSD（传统窄带语音编码，8 kHz）和 mSBC（宽带语音编码，16 kHz），实际选择取决于电脑蓝牙和音频系统以及耳机能力。

来源：[Microsoft 通信格式原文](../raw/microsoft/communications-audio-format-capabilities.md)。

## 适用范围

- 适用于使用传统蓝牙音频链路、同时包含播放和麦克风端点的耳机。
- 适用于 macOS 和 Windows 上“耳机只做输出正常、调用同一耳机麦克风后异常”的现象解释。
- 不适用于直接断言所有蓝牙耳机都只能单声道通话；LE Audio（新一代低功耗蓝牙音频）是另一套能力，且还依赖电脑、驱动、系统和耳机共同支持。

## 限制与反例

- Apple 页面没有直接给出 A2DP、HFP、编码器名称或具体采样率；这些名称和数值来自 Microsoft 的传统蓝牙资料以及 `MAC-home01-MacBookAir` 设备日志，不能把 Apple 的页面表述改写成 Apple 已公开确认这些字段。
- “切回 A2DP”不等于应用音频一定立即恢复。Microsoft 写的是麦克风关闭后系统会短暂切回 A2DP；`MAC-home01-MacBookAir` 的 2026-07-16 记录还显示应用可能处在重配置等待、静音插入或输出队列重建窗口。
- 有线耳机不需要在无线播放与无线双向通话链路之间切换，因此不能用有线耳机的正常结果反推蓝牙链路参数。

## 方案性质

- 把输入切到 Mac 内置麦克风、USB 麦克风或其他非目标蓝牙耳机：这是改变使用条件的绕过方案，可让目标耳机继续只做高质量输出。
- 语音结束后切换输入、再重新选择蓝牙输出：这是触发路由重建的临时恢复，不改变传统蓝牙 HFP 的能力上限。
- 更换支持合适 LE Audio 组合的耳机和电脑，或改用能同时呈现高质量输入输出的 USB/2.4G 接收器：这是改变底层链路的硬件方案，是否有效必须按具体设备组合验证。
