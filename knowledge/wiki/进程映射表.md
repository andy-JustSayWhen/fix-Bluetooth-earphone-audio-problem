# 进程映射表

## 用途
在开发本应用时，我发现无法根据某些进程名称推断出其对应的App等，所以，专门创设本文档用于记录某些进程的归属App、触发者App和触发实例。
仅当用户要求新增记录时，AI才可以在总表内创设新条目，否则只能维护已有条目。
## don't do
除非用户要求，否则：
不得修改总表的列数及列名称；
各级标题即本文档的目录树、文章结构（章节），你不得删增。比如目前有3个二级标题##，你删除了其中一个，或者新增了第四个##，这些都是 不被允许的。
不得罔顾章节的格式、结构，在你需要修改或删增条目时，已有条目就是最好的格式要求和格式示例。

## 触发实例

**实例 1：网易 UU远程触发 `replayd`**

| 字段 | 记录 |
| --- | --- |
| 宿主机 | `andymacbook-air` |
| 触发时间 | 2026-07-20 23:47:04 起 |
| 触发者 App | 网易 UU远程 |
| 触发场景 | 用户从外网 Windows（微软电脑系统）设备连接本机，网易 UU远程开始传输本机画面和系统播放声音。 |
| 调用关系 | 网易 UU远程 → `UURemoteServer`（网易 UU远程的后台服务进程） → `replayd`（macOS 系统采集服务） |
| `replayd` 的作用 | 代网易 UU远程执行画面与系统播放声音的底层采集。 |
| 与麦克风的关系 | 本次只确认到系统播放声音采集，没有证据表明 `replayd` 读取了 XIBERIA K03S 或其他蓝牙设备的实体麦克风。 |

**直接证据**

1. `replayd` 的可执行文件为 `/usr/libexec/replayd`，系统签名标识为 `com.apple.replayd`，签名链属于苹果。
2. 2026-07-20 23:47:04，`UURemoteServer` 记录外网 Windows 客户端已连接，随后开始画面采集和声音采集。
3. 同一时间，`UURemoteServer` 调用 `ScreenCaptureKit`（苹果提供的画面和系统声音采集能力）与 `ReplayKit`（苹果提供的录屏和共享能力）。
4. 2026-07-20 23:47:05，`replayd` 创建采样率为 44.1 kHz 的 `AudioTap`（把程序播放的声音接入采集流的系统通道）。
5. 2026-07-20 23:47:06 至 23:47:07，`replayd` 的系统权限请求连续指向进程号 `79812`。该进程号对应 `/Applications/UURemote.app/Contents/Helpers/UURemoteServer`，其父进程对应 `/Applications/UURemote.app/Contents/MacOS/UURemoteService -agent`。

**结论边界**

- 这条实例只证明本次网易 UU远程会话触发了 `replayd`，不能据此认定所有 `replayd` 活动都由网易 UU远程触发。
- `replayd` 出现在声音输入活动中，不等于它正在读取实体麦克风。
- 未确认实体麦克风归属时，不应将该活动归为麦克风占用，也不应因此终止或阻止 `replayd` 重新启动，否则可能中断网易 UU远程的画面或声音传输。
