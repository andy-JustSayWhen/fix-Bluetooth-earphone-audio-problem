# 蓝牙音频模式与参数变化

## 结论

本项目目前能可靠使用的判别顺序是：先看是否为传统蓝牙或 LE Audio，再看输入是否被打开，最后核对活动采样率、声道数和系统传输日志。采样率和声道数是可直接观测的设备层参数；编码器、传输链路名称和是否真正使用某个协议，只有在系统日志或设备诊断明确给出时才能写成直接事实。

## 传统蓝牙的参数表

| 模式 | 方向 | 本项目应关注的参数 | 来源明确内容 |
| --- | --- | --- | --- |
| A2DP（蓝牙立体声播放模式） | 主机到耳机 | 输出采样率、双声道、播放编码器 | 只负责输出；通常用于音乐、视频等普通播放 |
| HFP 宽带（蓝牙通话模式） | 耳机与主机双向 | 16 kHz、单声道、mSBC | 麦克风输入和单声道播放同时存在 |
| HFP 窄带 | 耳机与主机双向 | 8 kHz、单声道、CVSD | 设备或电脑不支持宽带时可能回落 |

来源：[Microsoft 传统蓝牙音频](../raw/microsoft/bluetooth-classic-audio.md) 与 [Microsoft 通信格式说明](../raw/microsoft/communications-audio-format-capabilities.md)。

Microsoft 还说明，Windows 会根据主机和耳机共同支持的编码器选择 A2DP 编码器；原文列出 SBC（基础蓝牙音频编码）、AAC、aptX Classic 和部分 Windows 11 设备上的 aptX Adaptive。这个列表不能用来判断本项目任一台 Mac 或某一款耳机当前实际使用的编码器。

## LE Audio 的参数边界

Bluetooth SIG 说明，Classic Audio（传统蓝牙音频）运行在 Bluetooth Classic 无线电上，LE Audio 运行在 Bluetooth Low Energy 无线电上；LE Audio 引入 LC3（低复杂度通信编码器）以及单播、广播等能力。Microsoft 的 Windows 资料进一步写明：

- Windows 11 的 LE Audio 通信能力取决于 LE Audio 耳机、电脑蓝牙硬件、驱动和音频系统共同支持。
- 使用 TMAP（电话与媒体音频配置）时，Windows 以 32 kHz 与设备传输通信音频；使用 HAP（助听器访问配置）时，Windows 以 16 kHz 或 24 kHz 传输，未来可能增加更多采样率。
- 一些 Windows 11 电脑可以在麦克风使用时保持 LE Audio 立体声；如果电脑或设备组合不支持，或用户配置为单声道，播放会变成单声道。

来源：[Bluetooth SIG LE Audio](../raw/bluetooth-sig/le-audio.md)、[Microsoft LE Audio](../raw/microsoft/bluetooth-low-energy-audio.md)、[Microsoft 通信格式说明](../raw/microsoft/communications-audio-format-capabilities.md)。

## 码率如何处理

目前不能从本项目的 macOS 监视器把“采样率”直接等同于“蓝牙码率”。`MAC-home01-MacBookAir` 上运行的监视器能读到设备端点的采样率和声道数；系统日志在部分实验中给出 `CVSD`、`mSBC`、`tsco`（蓝牙通话传输链路）和 `stereo=0`，但没有给出 Bose 或 K03S 当前音乐编码的精确比特率。

Bluetooth SIG 原文提到 LC3 在较低数据速率下仍能保持高质量，并引用了相对 SBC 低 50% 比特率的听测结论；这是该页面对 LC3 的介绍，不能拿来当 `MAC-home01-MacBookAir` 上耳机当时的实际码率。

## 模式变化的客观判据

在本项目中，下列组合只能写成“符合某模式特征”，除非系统日志同时点名协议：

- 约 44.1/48 kHz、双声道、没有启用目标蓝牙输入：符合传统蓝牙立体声播放特征。
- 约 8/16 kHz、单声道，同时存在目标蓝牙输入：符合传统蓝牙通话特征。
- 输入从其他设备切回目标蓝牙后，输出随之降为 16 kHz/单声道：说明设备层已经发生可见的格式变化。
- 仅有应用卡顿、无声或播放按钮延迟，而设备参数没有变化：不能据此声称协议再次切换；还需要应用音频日志或流级证据。

## 采样率与声道数的控制变量听感实验

如果要单独观察采样率和声道数对听感的影响，应把 Bose 保持在 A2DP（蓝牙立体声播放模式），输入固定为 Mac 内置麦克风，并使用同一个播放器、同一音量、同一段原音频。不要把“切换到 HFP”本身当成采样率实验，因为 HFP（蓝牙通话模式）还会同时改变声道、编码和传输链路。

本项目采用两组文件：

- 双声道 16 kHz 与 8 kHz：主要比较采样率变化。
- 单声道 16 kHz 与 8 kHz：更接近传统 HFP 的频率和声道限制。

建议按以下顺序做 A/B（两个版本轮流对比）听感比较：

| 对比 | 只改变的变量 | 重点听什么 |
| --- | --- | --- |
| 44.1/48 kHz 双声道 vs 16 kHz 双声道 | 采样率 | 高频亮度、镲片、空气感和细节 |
| 16 kHz 双声道 vs 8 kHz 双声道 | 采样率 | 闷感、清晰度和高频衰减 |
| 16 kHz 双声道 vs 16 kHz 单声道 | 声道数 | 左右分离、空间感、乐器定位 |
| 8 kHz 双声道 vs 8 kHz 单声道 | 声道数 | 声场收窄、声音拥挤程度 |
| 44.1/48 kHz 双声道 vs 16 kHz 单声道 | 综合差异 | 是否接近传统蓝牙通话听感 |

理论边界是：采样率降低后，能保留的最高频率大约不超过采样率的一半；声道从双声道变成单声道后，左右方向和空间信息会减少。对于本次背景音乐，若原音频高频内容或左右声道差异较少，差异可能比人声、镲片或明显左右移动的音效更难听出。

这组文件只能验证“采样率和声道数对听感的影响”，不能完整模拟 HFP。真实 HFP 还会涉及语音编码、蓝牙传输和音频流重建；因此 16 kHz/单声道 WAV 只是近似听感对照，不等于真实 HFP 输出。
