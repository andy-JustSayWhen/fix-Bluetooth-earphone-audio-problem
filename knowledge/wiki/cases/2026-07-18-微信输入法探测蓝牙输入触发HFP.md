# 2026-07-18：微信输入法探测蓝牙输入触发 HFP

## 实验环境

| 项目 | 记录值 |
| --- | --- |
| 环境 | `MAC-home02-MacMini` |
| 电脑 | Mac mini，Apple M1，机型标识 `Macmini9,1` |
| 操作系统 | macOS 15.7.7，系统内部版本 `24G720` |
| 蓝牙设备 | `Redmi电脑音箱-3899`，地址 `50:88:11:07:63:DA` |
| 候选程序 | 微信输入法 2.2.0（内部版本 617），程序标识 `com.tencent.inputmethod.wetype` |

## 触发时间线

| 时间 | 直接证据 |
| --- | --- |
| 09:14:36.195 | Redmi 输出短暂处于 `tacl`，即 A2DP；输入仍为 MOONDROP USB 设备 |
| 09:14:36.196 | `audioaccessoryd` 决定把 Redmi 设为临时默认输入 |
| 09:14:36.200 | Redmi 成为临时默认输入 |
| 09:14:36.187 至 09:14:36.241 | 微信输入法进程连续请求麦克风权限 |
| 09:14:36.254 | 微信输入法提交 `kBluetoothAudioDevicePropertyFormat request 0 -> 1` |
| 09:14:36.255 | 声音核心执行配置切换，选择 16 kHz、单声道通话端点 |
| 09:14:36.287 | 输出确认变为 `tsco`，即 HFP |

本窗口没有微信输入法的 `StartIO`（真正开始传输麦克风声音），因此不能把它描述成“正在录音”。本案例能确认的是：09:14:36.254 的格式请求由微信输入法进程提交，并立即导致 HFP。系统日志无法继续区分该请求是由蓝牙重连、快捷键、语音模块异常还是残留状态触发。

## 退出后的因果对照

1. 退出微信输入法主进程，确认没有自动重启。
2. 保持设备目标不变，断开并重新连接 `Redmi电脑音箱-3899`。
3. 最终默认输入仍为 Redmi，默认输出仍为 Redmi。
4. 从 2026-07-18 09:22:32 起连续六次读取：输入为 16 kHz、单声道；输出为 44.1 kHz、双声道。
5. 日志持续显示输出 `Current profile tacl`，没有微信输入法格式请求，也没有切回 HFP。

## 重新启动后的空闲对照

| 时间 | 操作与结果 |
| --- | --- |
| 2026-07-18 10:00:12 | 重新启动微信输入法，主进程号为 `65168` |
| 10:00:12 至 10:01:21 | 约 40 次连续读取均为 Redmi 输出 44.1 kHz、双声道和 A2DP |
| 10:00:12 至 10:01:25 | 无进程 `65168` 的蓝牙音频格式请求，无麦克风 `StartIO`，无 HFP 切换 |

该对照证明：仅启动微信输入法并保持空闲，不会必然触发 HFP。之前的“空闲时提交请求”不应当作已确认规律。

## 新增的快捷键线索

- 用户通过组合键启动微信语音输入，并将它绑定到一台会自动休眠的外置蓝牙小键盘。
- 小键盘唤醒后可能需要等待重连；用户在语音输入没有及时出现时会重复按键，并观察到语音面板刚出现又被取消。
- 异常期间，微信输入法菜单栏选项会明显变少，“语音输入（快捷键）”等选项消失；打开微信输入法应用和设置页后可暂时恢复。
- 本机键位映射配置还存在一条规则：把指定外设发出的“左 Option + A”转换为微信语音输入所用的“右 Option + 空格”。
- 上述信息只证明存在“外设唤醒、重复按键、转换后快捷键、微信语音模块”这条候选路径，尚无当时的原始按键事件，不能确认外置键盘是根因。

## 格式请求的正确解读

`0 -> 1` 和 `1 -> 0` 是系统内部格式索引或状态切换，不是两个独立的“请求 0”和“请求 1”，也不能当作通用的开关含义。实测中，`0 -> 1` 之后紧跟 HFP，`1 -> 0` 之后紧跟 A2DP 恢复；有些事件快速成对并自动恢复，另一些只有 `0 -> 1` 而缺少反向切换，因此长时间留在 HFP。为什么微信输入法有时没有及时提交反向切换，当前日志无法解释。

## 结论

在本机、本系统版本和本设备组合中，微信输入法进程是 09:14 这一次 HFP 接管的直接格式请求者。它当时没有真正开始录音，但仅启动进程并空闲也不会必然触发 HFP。触发该格式请求的上游条件仍待定位。

本案例同时证明：“同一 Redmi 蓝牙音箱同时作为默认输入和默认输出”不是充分原因。微信输入法退出后，相同默认输入输出条件可以稳定保持 A2DP。
