# 2026-07-16 采样率与声道听感对照

## 实验目标

在保持 Bose 为 A2DP（蓝牙立体声播放模式）输出的条件下，对同一条音频分别改变采样率和声道数，观察听感变化，避免把采样率、声道和 HFP（蓝牙通话模式）协议切换混为一个变量。

## 设备与固定条件

- 设备：本项目记录中的同一台 Mac 与 Bose QC Headphones。
- 默认输入：Mac 内置麦克风，不启用 Bose、DJI 或 K03S 蓝牙麦克风。
- 默认输出：Bose QC Headphones。
- 播放条件：同一个播放器、同一音量、关闭均衡器和其他音效。
- 原音频：[`趣味答题背景音乐_耳聆网_[声音ID：44249].mp3`](../../test/2026-07-16-采样率与声道听感对照/趣味答题背景音乐_耳聆网_[声音ID：44249].mp3)。原文件为 48 kHz、双声道、118.4 秒的 MP3。

## 测试产物

以下文件由同一原音频重新采样生成，均为未再压缩的 WAV（音频文件格式）：

| 文件 | 采样率 | 声道 | 用途 |
| --- | ---: | ---: | --- |
| [`16kHz-双声道`](../../test/2026-07-16-采样率与声道听感对照/趣味答题背景音乐-16kHz-双声道.wav) | 16 kHz | 2 | 只观察采样率变化 |
| [`8kHz-双声道`](../../test/2026-07-16-采样率与声道听感对照/趣味答题背景音乐-8kHz-双声道.wav) | 8 kHz | 2 | 只观察采样率变化 |
| [`16kHz-单声道`](../../test/2026-07-16-采样率与声道听感对照/趣味答题背景音乐-16kHz-单声道.wav) | 16 kHz | 1 | 近似传统 HFP 的频率和声道限制 |
| [`8kHz-单声道`](../../test/2026-07-16-采样率与声道听感对照/趣味答题背景音乐-8kHz-单声道.wav) | 8 kHz | 1 | 近似传统窄带通话的频率和声道限制 |

四个文件时长均为 118.4 秒。双声道版本用于隔离采样率变量；单声道版本用于观察声道减少后的综合听感。

## 控制变量对照

| 对比 | 只改变的变量 | 预期重点 |
| --- | --- | --- |
| 原音频 vs 16 kHz 双声道 | 采样率 | 高频细节、镲片、亮度、空气感 |
| 16 kHz 双声道 vs 8 kHz 双声道 | 采样率 | 8 kHz 更闷、更像电话，清晰度下降 |
| 16 kHz 双声道 vs 16 kHz 单声道 | 声道数 | 左右分离、空间感、乐器位置消失或变弱 |
| 8 kHz 双声道 vs 8 kHz 单声道 | 声道数 | 声场更集中，声音更拥挤 |
| 原音频 vs 16 kHz 单声道 vs 8 kHz 单声道 | 综合差异 | 与传统蓝牙通话听感的接近程度 |

## 证据边界

本记录目前保存了生成文件、固定条件和对照方法，尚未把用户的实际听感结果写成已验证结论。实际听感应在同一音量、随机播放顺序下记录，避免先知道文件参数后产生预期偏差。

这些 WAV 文件只改变采样率和声道数，不能完整复现 HFP 的语音编码、蓝牙传输和音频流重建过程。因此，16 kHz/单声道与 8 kHz/单声道只能作为近似听感参照，不能证明真实 HFP 使用了相同的音频编码。
