# 如何判定蓝牙音频设备的音频模式

## 定位

本文只规定“如何根据目标蓝牙设备最新的输出端点事实判定当前音频模式”。设备为什么进入 HFP 的原因判定见 [`../../knowledge/wiki/蓝牙音频设备进入HFP模式的原因.md`](../../knowledge/wiki/蓝牙音频设备进入HFP模式的原因.md)，一键修复流程见 [`HFP模式一键修复方案.md`](HFP模式一键修复方案.md)。

实现位置：

- [`features/bluetooth-audio-mode/index.ts`](../../tools/bluetooth-audio-mode-checker/features/bluetooth-audio-mode/index.ts)
- [`core/macos-audio-probe/index.ts`](../../tools/bluetooth-audio-mode-checker/core/macos-audio-probe/index.ts)
- [`core/macos-audio-events/index.ts`](../../tools/bluetooth-audio-mode-checker/core/macos-audio-events/index.ts)
- [`core/macos-bluetooth-link/index.ts`](../../tools/bluetooth-audio-mode-checker/core/macos-bluetooth-link/index.ts)

## 模式结果

- `HFP_HSP`：HFP/HSP 等低音质语音模式。
- `A2DP`：A2DP 等高音质播放模式。
- `UNKNOWN`：最新输出端点事实不足，或事实尚未组成上述任一完整条件。

## 唯一判定数据

数据来源：可用、标称和实际采样率及声道数来自 macOS Core Audio 输出端点属性（通过 Core Audio API 读取，非系统日志）输出端点，声音链路类型来自按蓝牙地址读取的系统蓝牙日志事件。

每台目标蓝牙设备只使用其最新输出端点的以下事实：

1. 可用采样率：输出端点向系统声明的全部可选标称采样率范围。
2. 标称采样率：系统当前为输出端点设置的目标采样率。
3. 实际采样率：输出端点当前时钟实际运行的采样率。
4. 输出声道数。
5. 声音链路类型：按蓝牙地址独立采集的该设备最新 `tsco` 或 `tacl` 事实。

三类采样率与声音链路类型是两组相互独立的证据。声音链路类型直接采用系统记录中能够归属到该设备蓝牙地址的最新链路事件。

## 判定顺序

按以下顺序命中即停止：

1. 该设备最新声音链路类型为 `tsco`：判定 `HFP_HSP`。
2. 输出可用采样率包含高于 `16 kHz` 的值，并且输出标称采样率或输出实际采样率至少一个不高于 `16 kHz`：判定 `HFP_HSP`。
3. 输出实际采样率高于 `16 kHz`，并且输出声道数不少于 2：判定 `A2DP`。
4. 其他情况：判定 `UNKNOWN`。

补充边界：

- `tsco` 是独立的 HFP/HSP 充分条件；即使采样率暂时已经变高，只要同一设备尚无更新的 `tacl` 事件覆盖最新 `tsco`，仍判定 `HFP_HSP`。
- `tacl` 不是 A2DP 的单独充分条件。只有“实际采样率高于 `16 kHz` 且输出声道数不少于 2”同时成立，才能判定 `A2DP`。
- 只有标称采样率高于 `16 kHz`、但实际采样率不可读或不高于 `16 kHz`，不得判定 `A2DP`。
- 输出可用采样率不包含高于 `16 kHz` 的值时，不能只凭低采样率判定发生了高音质向低音质的降级；若也没有 `tsco`，结果为 `UNKNOWN`。

## 声音链路事实的归属和时序

- `tacl/tsco` 必须按日志中的蓝牙地址归属到单台设备，不能把全局最后一条标记套给所有蓝牙设备。
- 只接受同一行同时包含可识别蓝牙地址和明确当前或启动链路类型的事件。
- 每个地址只保留时间最新的一条事件；旧事件不得覆盖新事件。
- 应用启动后持续监听新链路事件，并异步读取最近事件作为初始值；初始读取不得阻塞设备列表首次展示。
- 完整设备重扫后必须重新合并已保存的每设备最新链路事实，不能退回由模式推导链路类型。

## 实时刷新

- 输出标称采样率、实际采样率、声道或默认输出端点发生变化时，立即用最新事实重新判定并推送前端。
- 收到新的设备级 `tacl/tsco` 事件时，立即更新对应设备、重新判定并推送前端。
- 输入活动状态可以实时刷新页面上的“麦克风使用中”等角色，但不能直接改变模式结论。
- 手动刷新在服务端合并和限速；页面点击后立即恢复按钮，不设置固定冷却。
- 链路历史读取和完整设备扫描在后台执行，不得阻塞已有实时状态。

## 设备归组

同名输入端点和输出端点合并为同一设备卡，但模式只读取选中的输出端点事实。

输出端点优先级：当前默认输出 > 系统提示音输出 > 实际采样率和声道更高者。

输入端点只负责页面展示和声音路由选择，不参与模式判定。

同名系统输出端点只表示系统报告了输出方向端点，不代表设备具有物理扬声器。

## 输出给一键修复

模式判定完成后，向一键修复流程提供：

- 目标设备和输入、输出端点身份。
- 模式结果及判定时间。
- 当前默认输入、默认输出。
- 输出可用、标称、实际采样率和输出声道数。
- 按设备地址归属的最新声音链路类型及其时间。

## 页面展示

- 模式只显示在设备卡片顶部胶囊。
- `HFP_HSP` 显示 `HFP等模式（低音质语音模式）`；设备麦克风正在运行时可以追加“麦克风使用中”，但这只是运行状态。
- `A2DP` 显示 `A2DP等模式（高音质播放模式）`。
- `UNKNOWN` 显示 `模式无法确认`。
- 每张展开的设备卡片用一个设备级虚线框包住输入、输出端点框，标题显示独立采集的声音链路类型，并使用以下固定文案：
  - `声音链路类型：tacl（异步传输，用于单向音频播放）`
  - `声音链路类型：tsco（同步传输，常用于语音通话）`
  - 没有具体链路事实时显示 `声音链路类型：无法确认`。
- 输入、输出端点框分别展示“可用采样率、标称采样率、实际采样率、声道”；模式只读取输出端点框中的数据。
- 不在前端追加由规则拼接的长篇判定依据。

## 检查清单

- `tsco` 是否可以独立判为 `HFP_HSP`，且链路类型是否直接来自系统事件。
- 可用采样率包含高于 `16 kHz`，且标称或实际采样率不高于 `16 kHz` 时，是否判为 `HFP_HSP`。
- 是否只有“实际采样率高于 `16 kHz` 且输出声道数不少于 2”才判为 `A2DP`。
- 单声道高采样率、双声道低采样率、实际采样率缺失和证据矛盾时，是否不会误报 `A2DP`。
- 非默认设备是否直接使用该设备最新输出端点事实。
- 每设备链路事件是否按蓝牙地址和时间隔离。
- 具体链路类型后是否显示对应的固定中文解释，无法确认时是否不追加推断性解释。
- 前端是否随端点事实和链路事件立即更新，且刷新设备和切换设备没有变慢。
