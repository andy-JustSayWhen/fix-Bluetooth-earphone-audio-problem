# 蓝牙音频模式检查器

用于快速判定本机已连接的蓝牙音频设备当前属于哪种状态：

- A2DP（高音质立体声播放模式）；
- HFP/HSP（通话或耳麦模式）；
- LE Audio（新一代蓝牙音频模式）；
- 已连接但当前未使用；
- 参数不足，暂时无法可靠判定。

## 快速使用

可以在访达中直接双击本目录的 `run.command`。应用会启动并自动打开默认浏览器。

也可以在本目录运行：

```sh
./run.command
```

应用启动后会自动打开本地网页。页面会列出当前发现的蓝牙音频设备：

- 点击设备卡片，展开采样率、声道、输入输出状态和判定依据；
- 在设备列表上方直接选择系统默认扬声器或麦克风，选择后立即生效；
- 当前默认输出设备或采样率变化时，页面会自动更新，不必手动刷新；
- 麦克风占用识别只在设备进入低采样率模式后启动；占用结束即停止，不会在 A2DP 状态持续探测蓝牙输入；
- 设备显示 HFP/HSP 时，点击状态标签可启动一键修复；工具会先保存现场并判定本次原因，不会盲试全部方法；
- 连接新设备后，点击右上角“刷新设备”重新扫描；
- 页面没有发现设备时，会显示连接和刷新提示；
- 关闭启动应用的终端窗口即可停止应用。

工具直接读取并监听系统提供的音频设备信息；首次启动会自动编译随工具附带的本机监听组件，不需要另行安装依赖。网页只监听本机地址，不会向局域网或互联网开放。

## 一键修复

一键修复按以下顺序处理：

1. 目标已经恢复时立即结束。
2. 有程序实际读取目标麦克风时，只请求已确认的占用程序正常退出。
3. 没有占用时，用同一份系统声音事件先判断多端点会话，再判断格式请求。
4. 多端点会话会让用户选择保留输入还是输出；替代组合稳定只报告“绕过成功”。
5. 原因证据不足或原因处理后仍未恢复时，依次尝试非蓝牙输入中转和目标设备重连。
6. 首次恢复高采样率后，每 500 毫秒复查一次，连续三次稳定才报告“完全恢复”。

每个已确认进程在对应步骤只请求一次正常退出。进程未退出或后来重新出现时，工具不会再次结束、等待额外授权或启动后台阻止任务，而是继续执行后续固定步骤。

## 详细日志

详细日志默认启用，记录服务启停、网页请求、设备状态变化、麦克风占用、默认输入输出切换，以及一键恢复的诊断、每一步结果和子进程退出状态。日志采用 JSONL（每行都是一条可独立读取的结构化记录）格式，默认位置为 `logs/app.jsonl`。

- 单个文件达到 10 MB 后自动轮换；保留当前文件和 4 个历史文件；
- 密码、令牌、授权信息和 Cookie（网站保存的登录小票）等敏感字段自动隐藏；
- 可访问 `http://127.0.0.1:4173/api/logs/status` 查看是否启用、文件位置和当前大小；
- 如需临时关闭，可在启动前设置 `BLUETOOTH_AUDIO_LOG_ENABLED=0`。

## 判定原则

macOS（苹果电脑操作系统）通常不直接公开“当前协议名称”，因此工具同时展示系统报告的真实参数和推断结论：

- 读取设备支持的最高输出采样率，以及当前实际输出采样率；
- 当前默认输出设备，或正在实际采集声音的默认输入设备，会判定并显示活动参数；其他设备显示“活动参数未刷新”，不展示待机端点的采样率或声道；系统报告的同名输出端点不代表设备具有物理扬声器；
- 检测到麦克风被本机程序占用时，自动展开该设备卡片并收起其他卡片；
- 手动刷新由服务端合并与限速，页面不设置固定冷却等待；
- 设备支持高于 16 kHz、但当前实际输出不高于 16 kHz 时，判定为 HFP 等非 A2DP 模式；
- 当前实际输出高于 16 kHz 时，判定为 A2DP 模式；
- 如果无法读取当前实际输出采样率，或无法证明低采样率设备支持高于 16 kHz，则不强行判定；
- 声道、默认麦克风、蓝牙服务能力和系统链路日志均不参与模式判定；
- 当前默认输出设备或其实际采样率发生变化后，系统监听事件会立即触发重新判定并推送到页面；
- 设备没有承担当前输出时，不拿设备能力或待机格式冒充当前模式。

每次结果都会显示设备支持的最高输出采样率、当前实际输出采样率和所用规则，方便直接复核。
