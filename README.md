# fix-Bluetooth-earphone-audio-problem

本项目用于排查并解决 macOS 和 Windows 连接带麦克风的蓝牙耳机、蓝牙音箱等蓝牙音频设备时，播放音频期间出现的音质异常问题。

所谓“音频异常”通常有两类特征：

1. 音质很差：声音听起来空洞、发闷、失真、码率很低。
2. 传输不稳：声音断断续续，甚至突然没有声音。

你可能确信耳机没坏、系统设置没乱改，也反复在设置里切换输出和输入设备；有时能恢复，有时又完全没用。本项目把这些现象拆成可验证的协议、系统路由、应用占用和设备参数问题。

## 为什么

对于不支持 LE Audio 的蓝牙音频设备，日常播放音乐通常使用 A2DP 模式；当蓝牙麦克风被系统或应用调用时，设备可能被切到 HFP 模式。也就是说，很多听感上的“音频异常”本质上是 HFP 模式带来的质量降级。

A2DP 模式下，电脑可以向蓝牙耳机或音箱发送更高质量的音频，但它偏向单向播放，不用于传统蓝牙麦克风通话。

HFP 模式下，电脑为了支持实时双向通话，会牺牲播放质量和声道表现。

简单对比两种模式的常见音频参数：

- A2DP 的“高质量音频”一般是 44.1 kHz 或更高采样率，双声道，也就是常说的立体声。
- HFP 下的播放常见会降到 16 kHz，单声道。

如果把一段音乐从 44.1 kHz 降到 16 kHz，很多高频细节会消失，声音会明显发闷。如果再改成单声道，所有声音会挤到一起，空间感也会消失。

## 如何知道设备当前是否处于 HFP 模式

通常可以通过当前音频输入设备，也就是当前麦克风，初步判断蓝牙设备是否可能进入 HFP 模式。如果当前麦克风来自某个传统蓝牙音频设备，播放设备就有较高概率被切到 HFP。

macOS：

> 按下 option，然后点击顶部菜单栏的声音图标，展开声音输入列表。
> 或进入“系统设置 > 声音 > 输出与输入”。

Windows 11：

> 进入“设置 > 系统 > 声音 > 输入”。

需要注意：本项目实测发现，在 macOS 上选择蓝牙设备 A 作为输入、蓝牙设备 B 作为输出，也可能导致蓝牙设备 B 进入 HFP。因此，单靠“输入和输出分开指定为两个蓝牙设备”不一定能规避降级。

## 通过工具快速判定

仓库内已经提供[蓝牙音频模式检查器](tools/bluetooth-audio-mode-checker/)，可以查看目标设备当前处于 A2DP、HFP/HSP、LE Audio 或未使用状态。

可以在访达中双击工具目录内的 `run.command`，也可以进入工具目录后运行：

```sh
./run.command
```

应用会自动打开本地网页，展示蓝牙音频设备卡片。点击卡片可以展开判定依据；连接新设备后，点击页面右上角的“刷新设备”即可重新扫描。

## 怎么办

优先方案：

- 禁用或避开蓝牙麦克风。
- 选择 USB 麦克风或电脑内置麦克风作为输入。
- 使用 2.4G 接收器，而不是传统蓝牙通话链路。
- 购买前确认主机、系统和耳机都实际支持 LE Audio。

不可靠方案：

- 指定蓝牙 A 作为输入、蓝牙 B 作为输出。这个方案在部分系统和设备组合中仍可能触发 HFP。
