# fix-Bluetooth-earphone-audio-problem

本项目用于排查并解决macOS和Windows连接带麦克风的蓝牙耳机等蓝牙音频设备时，播放音频期间出现的音质异常问题。本项目所谓的“音频异常”有两大特征：

1.音质很差。声音听起来很空洞失真，

2.传输不稳。声音断断续续甚至突然没有声音

## 为什么

对于不支持LE Audio（蓝牙5.2起开始支持，但厂商也不一定搭载)的蓝牙耳机，日常会通过A2DP模式播放音频，若它的麦克风被调用，该设备会被强制切换到HFP模式。也就是说，是HFP模式导致的听感上的“音频异常”问题。

A2DP模式下，电脑可以向蓝牙耳机发送更高质量的音频，但传输模式是缓存式的、单向的，不支持麦克风调用，无法进行语音通话等行为。

HFP模式下，电脑通过牺牲音频质量，从而实现传输的实时性和双向性，因而解决了A2DP不能语音通话的问题。


我们简单对比两种模式下的音频参数，帮大家更好地理解这种听感上的区别，究竟是怎么形成的。

A2DP模式的“高质量音频”一般采样率为44.1kHz或者更高，声道是双声道（也即大家说的立体声）。

切换到HFP模式后，音频一般会降级到16kHz采样率，声道降为单声道。


如果我们把某条音频的采样率从44kHz降低到16kHz，听感上你会明显察觉到一些乐器听不见了、而且声音很闷，也就是丢了非常多的声音细节。如果这时候有画面，你会看到乐队里明明有些人在演奏沙锤、擦片这些乐器，却听不见它们的声音。

如果我们把某条音频的声道改成单声道，听感上所有乐器都还在，但你会觉得人声和所有乐器都挤在一起，你能听出来有哪些声音，但不知道它们分别是从哪发出的，就好像乐队突然从舞台各处集合到了舞台中央。


## 如何知道设备当前是否处于HFP模式

由于绝大部分的蓝牙耳机和音箱都不支持LE Audio，所以只要系统占用了某个蓝牙设备的麦克风，那么你的其他蓝牙设备都会被迫进入HFP模式。

通常我们可以通过直接查看当前电脑的音频输入设备（即当前麦克风）来判断耳机是否进入了HFP模式。如果当前麦克风来自某个蓝牙音频设备，那么所有的蓝牙设备一般都被切换到了HFP模式。

macOS：

> 按下option，然后点击顶部菜单栏的声音icon，展开声音输入列表。
> 或者，设置＞声音＞输出与输入

WIN 11：

> 设置＞系统＞声音＞输入


这里补充一下，理论上我选择蓝牙麦克风A作为输入，选择蓝牙耳机B作为输出，不会导致B也被迫进入HFP模式，但是WIN和macOS的工程师不会管那么多，只要有一个蓝牙设备进入了HFP，那么所有的蓝牙设备都会被切到HFP。

所以，你想要通过分别指定音频输入和输出设备，从而规避耳机进入HFP的想法并不可行。

而且，macOS还有个特性，当你选择某个蓝牙耳机作为输出，麦克风也会被自动切换到这个耳机，非常烦人。

## 通过这个工具快速判定

仓库内已经提供[蓝牙音频模式检查器](tools/bluetooth-audio-mode-checker/)，可以查看目标设备当前处于 A2DP、HFP/HSP、LE Audio 或未使用状态。

可以在访达中双击工具目录内的 `run.command`，也可以进入工具目录后运行：

```sh
./run.command
```

应用会自动打开本地网页，展示蓝牙音频设备卡片。点击卡片可以展开判定依据；连接新设备后，点击页面右上角的“刷新设备”即可重新扫描。

## 怎么办

禁用蓝牙麦克风

选择USB麦克风

插上2.4G接收器

购买支持LE Audio的蓝牙耳机

指定蓝牙A作为输入，蓝牙B作为输出 ❌
